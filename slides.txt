| Contexto, problemática y motivación |
+-------------------------------------+
Crecimiento del uso de IA/ML en finanzas: Las entidades financieras demandan cada vez más modelos de ML en producción. Sin embargo, llevar un modelo de laboratorio a la operación con control supone retos propios (gestión de datos, monitorización, mantenimiento continuo) que exceden el desarrollo tradicional de software
Brecha desarrollo-producción: Estudios indican que ~80 % de los proyectos de ML no llegan a producción, debido a problemas de reproducibilidad, gobernanza y escalabilidad
Necesidad operativa: Por ello, el objetivo es automatizar el ciclo de vida de los modelos para reducir la dependencia de tareas manuales y reforzar la trazabilidad. El TFM responde a esta motivación aplicando prácticas MLOps estandarizadas para pasar de prototipos aislados a modelos productivos fiables.

ODS 9: Industria, Innovación e Infraestructura
ODS 8: Trabajo decente y crecimiento económico

| Principios MLOps |
+------------------+
Stack de herramientas: Azure Repos, Azure DevOps, AzureML, Teradata, MLFlow
Flujo MLOps: Diagrama de flujo (Azure Repos -> Azure DevOps -> AzureML -> MLFlow -> Teradata -> Report)
Buenas prácticas:
- Separación de lógica de negocio y plataforma: separar el código del modelo de los componentes que orquestan su ejecución para reducir acoplamiento, facilitar pruebas y reutilizar infraestructura.
- Modularización y parametrización del código: dividir el pipeline en componentes con responsabilidad única y mover la configuración fuera del código para mejorar mantenibilidad y trazabilidad.
- Contratos explícitos y trazabilidad entre componentes: definir entradas, salidas y metadatos de cada componente, asociando versiones a cada ejecución para garantizar reproducibilidad y auditoría.
- Logging y evidencias de ejecución: registrar métricas, artefactos y configuraciones de cada ejecución para poder explicar desviaciones y reconstruir resultados.
